Gosh – The Unix shell reinvented

Gosh is a major departure from classic Unix shells that were centred around string manipulation and substitution. It is a modern, statically typed shell and scripting language.

  • Structured.
    Work with lists, tuples, functions and files, not just raw strings.
  • Functional in syntax and type system, but imperative for working with the file system.
  • Concise but not cryptic.
    No more  @du -hs "$v" >$@ or [[ -e "$f" ]]

Design

  • An expressive type system. Parametric polymorphism, algebraic types and typeclasses. Immutable data.
  • Programs are functions. Command line arguments and pipes are unified as parameters.
  • Files, globs and regexes are first-class features with syntax for literals.
  • Lists, tuples and dictionaries, with handling operations inspired by relational algebra.
  • A full scripting language with sane control flow, pattern matching and macros.

The language design has lots of open questions. Among them, how to:

  • Handle errors — exceptions, Maybe monad, other possibilites?
  • Dynamically type files — existential types?
  • Create a language that can be seamlessly used interactively, but also for safe scripting, for programming in the large.
  • Disambiguate the multiply operator (*) from a wildcard.

Getting it

You can download a precompiled Linux binary here: 
wget https://github.com/Fedjmike/gosh/releases/download/alpha/gosh
chmod +x gosh
sudo mv gosh /usr/local/bin

The source code is hosted on GitHub, released under the GPL. Call make to build it. Caveat: you won't actually be able to because it relies on a bunch of libraries.

Development

Development began in July 2015. Gosh is written in C11 and tested on Linux. Currently, it uses the Boehm garbage collector and GNU Readline.

Future

There's a lot for us to implement. The repo contains a todo list which will help you find out what's been done and what's there is to do.

Beyond the language itself, there's:

  • A C API for writing native Gosh programs.
  • Better interactive features. Syntax highlighting, intelligent completion, history autocompletion, job control.
  • Support for BSD, OS X and Windows.
  • JIT compilation?
